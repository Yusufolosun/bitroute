# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ----------------- |
| 1.x     | :white_check_mark: |
| <1.0    | :x:               |

## Reporting a Vulnerability

If you discover a security vulnerability in BitRoute, please report it by emailing:

**security@bitroute.io**

(If this address is not set up, open a private GitHub issue instead.)

Please include:
- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Your contact information

We will respond as quickly as possible and keep you informed of progress.

## Disclosure Policy

- We will investigate all reports promptly
- We will not disclose vulnerabilities until a fix is released
- Credit will be given to reporters if desired

## Security Best Practices

- Never share your seed phrase or private keys
- Use hardware wallets for mainnet
- Always verify contract addresses
- Keep dependencies up to date

## Acknowledgements

Thank you for helping keep BitRoute and its users safe!
